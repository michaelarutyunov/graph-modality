ONTOLOGY
========

Entity types:

- Construct: a bipolar cognitive dimension used to evaluate the domain under
  discussion. Each pole must be independently grounded in the transcript:
  grounding_spans_positive lists verbatim [Human] phrases supporting the positive
  pole; grounding_spans_negative lists phrases supporting the negative pole. Set
  bipolarity_complete to true ONLY when both lists are non-empty. If only one pole
  is grounded, leave the other list empty and set bipolarity_complete to false.
  Do NOT invent the opposite pole — every pole must be traceable to a [Human] turn.

- Value: a terminal motivational state that constructs serve. High-abstraction,
  representing what the respondent ultimately cares about. The number of distinct
  grounding_spans reflects the value's salience in the interview.

- Stance: a valenced attitude position toward some aspect of the domain. Required
  attribute: valence (see Valence Taxonomy below). grounding_spans may include
  multiple spans if the stance recurs across turns.

- CognitiveStyleMarker (CSM): a stable processing tendency — HOW the person
  reasons, not WHAT they care about. To qualify as a CSM, the pattern must appear
  across at least two different [Human] turns (recurrence as evidence of "stable
  tendency"). Each CSM's grounding_spans must include at least one span from each
  of two distinct turns. There is no fixed ceiling on CSM count; extract every
  pattern that meets the recurrence test.

Valence taxonomy (Stance nodes only):

  positive  — favourable disposition toward the object
  negative  — unfavourable disposition toward the object
  mixed     — different valence toward different aspects of the same object
              (e.g., positive about AI for routine tasks, negative for creative work)
  ambivalent — simultaneous conflicting valence toward the same aspect
              (e.g., "I trust it and I don't trust it at the same time")

  When uncertain, prefer "mixed" over "ambivalent" — ambivalent requires evidence
  of simultaneous conflict, not merely different attitudes toward different facets.

Relation types:

  SERVES         Construct → Value
    The construct is instrumental to achieving the value.
    Include a one-line rationale.

  EXPRESSED_VIA  Stance → Construct
    The stance is expressed through this evaluative dimension.
    Include a one-line rationale.

  MODULATED_BY   Construct → CognitiveStyleMarker
                 OR Stance → CognitiveStyleMarker
    The construct or stance is shaped by a cognitive processing tendency.
    Include a one-line rationale.

  CONFLICTS_WITH Construct ↔ Construct (undirected)
    Use ONLY when the respondent explicitly expresses tension or contradiction
    between the two constructs. The rationale MUST quote the [Human] span that
    shows the conflict.

  SUBSUMES       Value → Value (directed: more specific → broader)
    One value is a specific instance or component of a broader value.
    This relation is INFERRED (respondents rarely state value hierarchies
    explicitly). Set grounding to "inferred" and make the rationale cite the
    grounding spans of BOTH values that license the inference.

  IMPLIES        Construct → Construct (directed)
    One construct logically entails or presupposes the other.
    This relation is INFERRED. Set grounding to "inferred" and make the
    rationale cite the grounding spans of BOTH constructs that license the
    inference.

CONSTRAINTS
===========

1. Extract nodes from [Human] turns only. Use [AI] turns as context only.

2. Every node must include verifiable grounding:
   - Constructs: grounding_spans_positive (list) AND grounding_spans_negative
     (list). bipolarity_complete is true ONLY when both lists are non-empty.
   - Values, Stances, CSMs: grounding_spans (list of verbatim [Human] phrases).
     A single-element list is fine when the entity appears once; longer lists
     reflect higher salience.

3. Every edge must include a rationale (one-line explanation of why the relation
   holds). For CONFLICTS_WITH edges, the rationale MUST quote the [Human] span
   demonstrating the conflict.

3a. Every edge must include a grounding field, one of:
   - "explicit": the relation itself is stated or directly quotable in a [Human]
     span; the rationale quotes that span.
   - "inferred": the relation is your inference, licensed by the endpoint nodes'
     grounding spans; the rationale must cite the grounding spans of BOTH endpoint
     nodes that license it.
   SUBSUMES and IMPLIES are always "inferred". CONFLICTS_WITH is always "explicit".
   Do NOT invent relations you cannot trace to grounded spans on both endpoints.

4. No direct Stance → Value edges. Stances must connect to Values via Constructs.

5. CognitiveStyleMarkers: extract every pattern that meets the recurrence test
   (spans from at least two different [Human] turns). There is no ceiling on CSM
   count — do NOT cap at 2.

6. Extract every node and edge that meets the above criteria. Do not artificially
   limit counts. If the transcript contains no evaluative content about the
   domain, it is acceptable and correct to return zero nodes and zero edges.

7. Discard any node you cannot ground in at least one [Human] turn.
